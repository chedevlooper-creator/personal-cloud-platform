## 2. Color System

All tokens are defined in `apps/web/src/app/globals.css` using OKLCH. **Never use raw hex in components** — always use semantic token names via Tailwind CSS variables (`bg-background`, `text-primary`, etc.).

### Semantic Tokens (reference names)

| Token                   | Dark OKLCH                  | Light OKLCH                 | Usage                                  |
| ----------------------- | --------------------------- | --------------------------- | -------------------------------------- |
| `--background`          | `oklch(0.13 0.008 260)`     | `oklch(0.985 0.002 240)`    | Page/app background                    |
| `--foreground`          | `oklch(0.93 0.005 240)`     | `oklch(0.15 0.01 250)`      | Primary text                           |
| `--card`                | `oklch(0.17 0.01 260)`      | `oklch(1 0 0)`              | Card/panel surfaces                    |
| `--primary`             | `oklch(0.65 0.16 240)`      | `oklch(0.55 0.15 240)`      | CTA buttons, active states, links      |
| `--muted`               | `oklch(0.22 0.01 260)`      | `oklch(0.955 0.005 240)`    | Subtle backgrounds, tags               |
| `--muted-foreground`    | `oklch(0.6 0.01 250)`       | `oklch(0.5 0.01 250)`       | Secondary/helper text                  |
| `--border`              | `oklch(0.25 0.01 260)`      | `oklch(0.91 0.005 250)`     | Dividers, input borders                |
| `--destructive`         | `oklch(0.65 0.2 25)`        | `oklch(0.577 0.245 27.325)` | Error states, delete actions           |
| `--ring`                | `oklch(0.65 0.16 240)`      | `oklch(0.55 0.15 240)`      | Focus rings (3px, 50% opacity)         |
| `--sidebar`             | `oklch(0.15 0.009 260)`     | `oklch(0.975 0.003 240)`    | Sidebar background                     |

### Accent Glow Palette (decorative only, `aria-hidden`)

Used for radial gradient blobs in hero/chat surfaces — **never for conveying information**.

| Name        | Color approx    | Usage                                 |
| ----------- | --------------- | ------------------------------------- |
| Violet      | `#B85CFF`       | AI/intelligence metaphor, top-right   |
| Amber       | `#F5A524`       | Warmth, bottom-left                   |
| Ice Blue    | `#3FB6E0`       | Technology, left diagonal             |
| Mint        | `#7CD992`       | Success/live indicators               |

### Chart Colors (sequential, accessible)

```
chart-1 (blue-teal) → chart-2 (mint green) → chart-3 (amber) → chart-4 (violet) → chart-5 (orange)
```

All chart color pairs against `--background` meet ≥ 3:1 contrast. Do not use red/green alone for status — always pair with an icon or label.

### Rules
- `color-semantic`: Always use token names; no inline hex/oklch in JSX
- `color-dark-mode`: Dark surfaces use desaturated tonal variants (chroma ≤ 0.01 for neutrals)
- `color-not-decorative-only`: Status colors (destructive/success) must include icon + text, not color alone
- `dark-mode-pairing`: Light and dark variants are already paired; test contrast in both before shipping

## 3. Typography

### Font Stack

| Role       | Font            | Variable         | Weights     |
| ---------- | --------------- | ---------------- | ----------- |
| Body/UI    | Inter           | `--font-sans`    | 400, 500, 600, 700 |
| Code       | JetBrains Mono  | `--font-mono`    | 400, 500    |
| Heading    | Inter (same)    | `--font-heading` | 600, 700    |

> Both fonts are loaded via `next/font/google` with `latin` subset and CSS variable injection. No fallback flash (FOIT avoided via `font-display: swap` behavior in Next.js font system).

### Type Scale

| Step  | Size     | Weight | Line-height | Use                                       |
| ----- | -------- | ------ | ----------- | ----------------------------------------- |
| `xs`  | 12px     | 400    | 1.5         | Timestamps, metadata badges               |
| `sm`  | 14px     | 400    | 1.5         | Secondary labels, table cells, captions   |
| `base`| 16px     | 400    | 1.625       | Body text, form inputs, descriptions      |
| `lg`  | 18px     | 500    | 1.5         | Card titles, section intros               |
| `xl`  | 20px     | 600    | 1.4         | Page section headings                     |
| `2xl` | 24px     | 600    | 1.3         | Panel/modal headings                      |
| `3xl` | 30px     | 700    | 1.2         | Page titles (dashboard "Home")            |
| `4xl` | 36px     | 700    | 1.15        | Hero/landing large text                   |

### Rules
- Minimum 16px for all body text on mobile (avoids iOS auto-zoom)
- Use `font-tabular-nums` (tabular figures) for file sizes, timestamps, and numeric data columns
- Line length: 60–75 chars max for long-form content (prose in docs/settings descriptions)
- `weight-hierarchy`: headings 600–700, body 400, labels/captions 500
- Never use `text-xs` (12px) for anything the user must read without deliberate action

## 4. Layout & Spacing

### Spacing System

Follows 4px/8px incremental grid (Tailwind `spacing` scale, 1 unit = 4px).

| Use case                   | Value   | Tailwind class  |
| -------------------------- | ------- | --------------- |
| Icon-to-text gap           | 8px     | `gap-2`         |
| Input inner padding        | 12px    | `px-3`          |
| Card padding               | 16px    | `p-4`           |
| Section spacing (vertical) | 24–32px | `space-y-6/8`   |
| Page content inset         | 24px    | `p-6`           |
| Sidebar width (expanded)   | 240px   | `w-60`          |
| Sidebar width (collapsed)  | 56px    | `w-14`          |
| Modal max-width            | 480px   | `max-w-lg`      |
| Main content max-width     | 1280px  | `max-w-7xl`     |

### Breakpoints

| Name    | Pixel  | Behavior                                              |
| ------- | ------ | ----------------------------------------------------- |
| default | < 768  | Mobile: bottom drawer, collapsed sidebar              |
| `md`    | 768px  | Tablet: sidebar as overlay                            |
| `lg`    | 1024px | Desktop: persistent sidebar, main canvas expands      |
| `xl`    | 1280px | Wide: more whitespace, wider panels                   |

### Z-index Scale

| Layer           | Value | CSS var (suggested)  |
| --------------- | ----- | -------------------- |
| Base content    | 0     | —                    |
| Raised cards    | 10    | `--z-raised`         |
| Sticky headers  | 20    | `--z-sticky`         |
| Sidebar overlay | 40    | `--z-sidebar`        |
| Modals/drawers  | 100   | `--z-modal`          |
| Toasts          | 200   | `--z-toast`          |
| Command palette | 500   | `--z-command`        |
| Tooltips        | 1000  | `--z-tooltip`        |

### Rules
- `viewport-meta`: `width=device-width, initial-scale=1` — already in `apps/web/src/app/layout.tsx`; **never disable zoom**
- `mobile-first`: Design starts at 375px; scale up
- `fixed-element-offset`: Fixed navbar/bottom bar must reserve body padding to avoid content overlap
- `viewport-units`: Use `min-h-dvh` not `min-h-screen` / `100vh` to account for mobile browser chrome
- `safe-area-awareness` (web): Not critical but respect `env(safe-area-inset-*)` in any fixed-position elements

## 6. Visual Style & Effects

### Primary Style: Minimal Dark + Selective Glassmorphism

The app uses **minimal dark** as the base visual language. Glassmorphism (`backdrop-blur`, `bg-*/XX%`, `border-white/[0.07]`) is applied **selectively** to:
- Toast / notification overlays
- Modal scrim layers
- Floating command palettes
- Sidebar overlay (mobile)

**Never** use `backdrop-blur` on primary content cards or page sections — it degrades performance and adds visual noise.

### Surface Elevation

| Surface         | Background token   | Shadow / Effect                                     |
| --------------- | ------------------ | --------------------------------------------------- |
| Page bg         | `background`       | None                                                |
| Sidebar         | `sidebar`          | Right border: `border-border`                       |
| Card / Panel    | `card`             | `shadow-sm` in light; subtle border in dark         |
| Input           | `background`       | Border `border-input`, focus ring `ring-2 ring-ring/50` |
| Dropdown        | `popover`          | `shadow-lg rounded-lg border border-border`         |
| Modal           | `card`             | `shadow-2xl` + `backdrop-blur-sm` on scrim          |
| Toast           | Zinc-950/80 glass  | `backdrop-blur-sm shadow-[0_8px_24px_-12px_...]`    |
| Command palette | `card`             | `shadow-2xl rounded-2xl border border-border`       |

### Decorative Glows (aria-hidden only)

The chat/home surface uses radial gradient blobs. Rules:
- Always `aria-hidden="true"` and `pointer-events-none`
- Max opacity: violet 12%, blue 7%, amber 9%, mint 8%
- Blur: `blur-3xl`
- Don't add glows to utility pages (files, settings, terminal)

### Dot/Grid Background

`DottedBackground` component using `14px × 14px` grid lines. Apply on:
- Chat/home hero area (opacity 45%)
- NOT on dense data pages (files table, automations list)

### Icon System

- **Library**: Lucide React (already in use) — stroke icons, consistent 24px default
- **Size tokens**: `size-3` (12px captions), `size-4` (16px default), `size-5` (20px actions), `size-6` (24px nav)
- **Stroke width**: 1.5px default (Lucide standard) — never mix with other icon libraries
- No emoji as structural icons
- Navigation items require both icon AND text label (icon-only fails discoverability)

## 7. Animation & Motion

### Duration Tokens (apply via Tailwind `duration-*`)

| Type               | Duration  | Easing           |
| ------------------ | --------- | ---------------- |
| Micro (hover, tap) | 150ms     | ease-out         |
| Standard           | 200ms     | ease-out         |
| Emphasized         | 300ms     | ease-in-out      |
| Complex/enter      | 300–400ms | ease-out         |
| Exit               | 150–200ms | ease-in (faster) |

Exit animations must be 60–70% of enter duration.

### Approved Motion Patterns

| Pattern         | Implementation                                           |
| --------------- | -------------------------------------------------------- |
| Page enter      | `animate-in fade-in slide-in-from-bottom-4 duration-300` |
| Modal enter     | `animate-in fade-in zoom-in-95 duration-200`             |
| Modal exit      | `animate-out fade-out zoom-out-95 duration-150`          |
| Toast enter     | `animate-in fade-in slide-in-from-top-1 duration-200`    |
| Sidebar slide   | `transition-[width] duration-200 ease-out`               |
| Button press    | `active:translate-y-px` (1px down on press)              |
| Skeleton shimmer| Tailwind `animate-pulse` on muted bg                     |

### Rules
- Only animate `transform` and `opacity` — never `width`, `height`, `top`, `left` (causes layout reflow)
- `reduced-motion`: All animations must be disabled or minimized via `prefers-reduced-motion` — use `motion-safe:` prefix on animation classes
- Max 1–2 animated elements per view (no animation chaos on data pages)
- Skeleton loading for any fetch > 300ms; spinner only for user-triggered actions

## 8. Component Patterns

### Buttons

| Variant     | Use                                          | Anti-pattern                        |
| ----------- | -------------------------------------------- | ----------------------------------- |
| `default`   | Primary CTA (one per page section)           | Multiple primary buttons in a row   |
| `outline`   | Secondary actions                            | —                                   |
| `ghost`     | Toolbar/icon actions, nav items              | —                                   |
| `destructive` | Delete / danger (visually separated)       | Red button inline with normal flow  |

- Each screen should have **one** primary CTA; others are secondary/ghost
- Disabled state: `opacity-50 pointer-events-none` (already in CVA config)
- Loading state: disable button + show spinner during async operations
- Min touch target: 44px height — use `h-9` (36px) minimum and `py-2 px-4` for comfortable targets

### Forms

- Every input must have a `<label>` — no placeholder-only labels
- Error messages: `text-destructive text-sm mt-1` below the affected field
- Helper text: `text-muted-foreground text-sm mt-1` below the field
- Required marker: `*` in label, `text-destructive`
- Multi-step flows: show step progress bar or step indicator
- Long forms: auto-save drafts if possible; warn before unload

### Navigation (App Shell)

The main navigation is a **left sidebar** with icon + label items.

| Nav item        | Route           | Icon (Lucide)   |
| --------------- | --------------- | --------------- |
| Home/Chat       | `/dashboard`    | `MessageSquare` |
| Files           | `/files`        | `Folder`        |
| Automations     | `/automations`  | `Zap`           |
| Terminal        | `/terminal`     | `Terminal`      |
| Hosting         | `/hosting`      | `Globe`         |
| Snapshots       | `/snapshots`    | `Camera`        |
| Settings        | `/settings`     | `Settings`      |

Rules:
- Active nav item: highlight with `bg-sidebar-accent text-sidebar-primary-foreground` 
- Collapsed sidebar: show icon only (tooltip with label on hover)
- Mobile: sidebar becomes a drawer/sheet (not bottom nav — this is a desktop-first app)
- Dangerous actions (sign out) visually separated at bottom of sidebar

### Modals & Dialogs

- Use `Dialog` from shadcn/ui for confirmations and focused flows
- Always include a close button with `aria-label="Close"`
- Scrim: `bg-black/50` minimum for legibility
- Destructive confirmation dialogs: red destructive button, separated from Cancel
- Swipe-down / Escape key must dismiss

### Toast Notifications

- Auto-dismiss: 4–5 seconds for info/success; 8 seconds for errors (user needs time to read)
- `aria-live="polite"` on toast region (already standard in shadcn Toaster)
- Never steal focus
- Position: top-center on desktop, top-center on mobile

## 9. Accessibility Baseline

Every component shipped must pass these checks:

| Check                  | Standard                                              |
| ---------------------- | ----------------------------------------------------- |
| Text contrast          | ≥ 4.5:1 for body text; ≥ 3:1 for large text (24px+)  |
| Icon contrast          | ≥ 3:1 against background                             |
| Focus ring             | Visible, 2–4px, `ring-ring/50` — already in globals  |
| Keyboard navigation    | Tab order matches visual order; all interactive reachable |
| Screen reader labels   | `aria-label` on icon-only buttons; `alt` on images   |
| Color-not-only         | Status always has icon + text, not color alone        |
| Skip link              | "Skip to main content" — already in `layout.tsx`     |
| Heading hierarchy      | h1 → h2 → h3 sequential, no level skips              |
| Reduced motion         | Wrap all animation classes with `motion-safe:`        |
| Form labels            | `<label htmlFor>` or `aria-label` on every input      |

### Contrast Reference (Dark Mode)

| Pair                                    | Ratio  | Result |
| --------------------------------------- | ------ | ------ |
| `--foreground` on `--background`        | ~14:1  | ✅ AAA |
| `--muted-foreground` on `--background`  | ~5.5:1 | ✅ AA  |
| `--primary` on `--background`           | ~4.8:1 | ✅ AA  |
| `--destructive` on `--background`       | ~4.5:1 | ✅ AA  |
| `--muted-foreground` on `--card`        | ~4.1:1 | ⚠️ verify per use |

## 10. Page Patterns

### Dashboard / Home (Chat Surface)
- Decorative glows + dot background ON
- Hero greeting centered, large (3xl–4xl)
- Single primary input (chat bar) — the dominant CTA
- Quick-action chips below input (secondary)
- Tool status toast anchored above input

### File Manager
- Dense data table layout
- Breadcrumb navigation for folder hierarchy
- Batch selection with action toolbar
- No decorative glows
- Sort, filter, upload as primary actions

### Chat / Agent Conversation
- Message bubbles full-width container, max `max-w-3xl` centered
- AI messages: left-aligned, `card` background
- User messages: right-aligned, `primary` tint background
- Code blocks: JetBrains Mono, `bg-muted` surface, copy button
- Tool-call cards: collapsible, `border border-border rounded-xl`
- Streaming: cursor/spinner in last message, not a separate loading state

### Terminal
- Full-height xterm.js canvas, no padding steal from terminal area
- Dark bg forced even in light mode (terminal UX convention)
- Monospace throughout
- Status bar at bottom: connection state, session info

### Settings
- Sidebar sub-navigation on desktop (left), tabs on mobile
- Section cards with clear h2 headings
- Destructive zone (delete account, revoke tokens) visually separated with `border-destructive` card

## 11. Anti-Patterns (Avoid)

| Anti-pattern                           | Reason                                              |
| -------------------------------------- | --------------------------------------------------- |
| Emoji as icons (`🚀`, `⚙️`)           | Not scalable, inconsistent, can't theme             |
| Raw oklch/hex values in JSX            | Breaks theme system, fails dark mode                |
| `backdrop-blur` on primary content     | Performance hit, adds noise on data pages           |
| Multiple primary buttons per section   | Dilutes user focus                                  |
| Placeholder-only form labels           | Fails accessibility                                 |
| `w-screen` or `100vw` containers       | Causes horizontal scroll on mobile                  |
| `min-h-screen` / `100vh`               | Broken on mobile browsers — use `min-h-dvh`         |
| Animating `width`, `height`, `top`     | Causes layout reflow, drops fps                     |
| Color-only status indicators           | Fails colorblind users                              |
| Icon-only navigation items             | Fails discoverability; always pair with text label  |
| `z-index` values without scale         | Creates z-index wars; use the scale above           |
| Decorative glows on data-dense pages   | Visual noise on files/automations/settings          |
| Animations without `motion-safe:`      | Breaks reduced-motion accessibility preference      |

## 12. Pre-Delivery Checklist

Before shipping any UI component or page:

### Visual
- [ ] No emoji icons — Lucide SVG only
- [ ] All colors reference semantic tokens (no inline hex/oklch)
- [ ] Dark and light mode tested (both pass contrast ≥ 4.5:1 for body text)
- [ ] Border radius follows the radius scale
- [ ] Elevation/shadow consistent with Surface Elevation table

### Interaction
- [ ] Primary CTA count per section ≤ 1
- [ ] All buttons show loading state during async ops
- [ ] Disabled states use `opacity-50 pointer-events-none` + semantic `disabled` attribute
- [ ] Destructive actions gated by confirmation dialog or require two-step confirmation
- [ ] Toast notifications auto-dismiss (4–5s info, 8s error)

### Accessibility
- [ ] Contrast ≥ 4.5:1 for body text in both themes
- [ ] Focus rings visible on all interactive elements
- [ ] `aria-label` on all icon-only buttons
- [ ] `alt` text on all meaningful images
- [ ] Keyboard navigation works in logical tab order
- [ ] Animation classes wrapped in `motion-safe:`
- [ ] Form inputs have associated `<label>` elements

### Layout
- [ ] No horizontal scroll at 375px viewport
- [ ] Content not hidden behind fixed headers/sidebars (correct padding offsets)
- [ ] `min-h-dvh` used instead of `min-h-screen`
- [ ] Spacing follows 4/8px rhythm (no arbitrary values)

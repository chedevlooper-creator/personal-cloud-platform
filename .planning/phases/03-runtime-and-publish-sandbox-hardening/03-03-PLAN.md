# Plan 03-03: Verify Hosted Secret Handling And Sandbox Regressions

## Objective

Close Phase 3 by proving sandbox and hosted-service secret contracts survive
regressions.

## Target Files

- `services/publish/src/service.test.ts`
- `services/publish/src/routes.test.ts`
- `services/runtime/src/provider/docker.test.ts`
- `services/runtime/src/service.test.ts`
- `services/agent/src/tools/run_command.ts`
- Phase 3 GSD summary and verification artifacts

## Must Haves

- Hosted-service env vars remain encrypted at rest and masked in API-facing
  responses.
- Invalid env names never reach Docker `Env`.
- Sandbox tests fail if root execution, privileged mode, capability defaults,
  resource limits, security opts, image policy, or network policy regress.
- Phase 3 verification documents any remaining production sandbox debt.

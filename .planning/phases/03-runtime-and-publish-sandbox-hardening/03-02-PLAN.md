# Plan 03-02: Add Image And Execution Policy Visibility

## Objective

Reject unapproved runtime/publish images before Docker launch and expose
runtime command policy details where tools and terminal flows can show them.

## Target Files

- `services/runtime/src/service.ts`
- `services/runtime/src/service.test.ts`
- `services/runtime/src/provider/types.ts`
- `services/agent/src/tools/run_command.ts`
- `services/agent/src/tools/run_command.test.ts` if needed
- `services/publish/src/service.ts`
- `services/publish/src/service.test.ts`

## Must Haves

- Runtime creation validates images against an allow-list before inserting or
  creating containers.
- Publish service validates its internal image choice against the same policy
  concept before Docker launch.
- Runtime command policy exposes timeout, network, approval, and blocked command
  categories in a structured constant or method.
- Agent `run_command` tool definition includes policy wording sourced from that
  policy rather than stale free-text duplication.
- Tests cover rejection paths and policy metadata without Docker.

## Verification Commands

```powershell
pnpm --filter @pcp/runtime-service test
pnpm --filter @pcp/publish-service test
pnpm --filter @pcp/agent-service test
pnpm test
pnpm typecheck
```

## Done When

- Unapproved images cannot reach Docker.
- Users and agents can see meaningful command policy details before side
  effects.

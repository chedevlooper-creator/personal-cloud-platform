</objective>

<must_haves>
## Must Haves

### Truths
- Logs must not include PII, API keys, session cookies, provider keys, or
  decrypted secrets.
- Pino/ Fastify logging should include `correlationId`, `userId`, and `service`
  where available.
- Health checks should remain lightweight and suitable for local smoke and
  deployment probes.
- Graceful shutdown should not create destructive behavior or kill unrelated
  user processes.

### Phase Success Criteria Covered
- Logs include required context fields and redact secrets/PII.
- Health checks and shutdown behavior are normalized enough for later CI/smoke
  and deployment work.
</must_haves>

<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Secrets or PII appear in logs | High | Configure redaction and avoid logging raw bodies/errors. |
| Requests cannot be correlated during incident review | Medium | Ensure correlation IDs are generated/propagated. |
| Service reports healthy while critical config is unsafe | Medium | Health/startup checks reflect parsed config state. |
| Shutdown drops active work unpredictably | Medium | Add bounded graceful shutdown hooks where appropriate. |
</threat_model>

<tasks>
## Tasks

<task id="1" type="analysis">
Inventory current logging, health, and shutdown behavior in all services. Note
where `console.error` is used and where pino request context is missing.
</task>

<task id="2" type="implementation">
Add or normalize request correlation handling so logs can include
`correlationId`, `userId`, and `service` without leaking sensitive values.
</task>

<task id="3" type="implementation">
Add safe pino redaction for cookies, authorization headers, provider keys,
encrypted/decrypted secrets, and obvious credential fields.
</task>

<task id="4" type="implementation">
Normalize health routes and startup readiness semantics across services without
adding heavy dependency checks to every request.
</task>

<task id="5" type="implementation">
Add bounded graceful shutdown handling where missing, especially for services
with workers, WebSockets, or container/session resources.
</task>

<task id="6" type="test">
Add focused tests or smoke assertions for representative health behavior and
log redaction where practical.
</task>

<task id="7" type="docs">
Update production docs only if service health/shutdown expectations or required
environment behavior changes.
</task>
</tasks>

<verification>
## Verification

- Run `pnpm typecheck`.
- Run targeted tests for services touched by logging/health/shutdown changes.
- Run `pnpm smoke:local` only if local services are running or can be started
  safely in the current environment.
- Review logs from at least one representative request path to confirm secrets
  are not printed.
</verification>

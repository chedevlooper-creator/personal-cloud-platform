# Plan 01-01: Centralize Session And Env Validation

<objective>
Create a shared, fail-closed foundation for session validation and service env
validation without changing the product's cookie-session auth model.
</objective>

<context>
Read before editing:
- `.planning/phases/01-contracts-config-and-auth-foundation/01-CONTEXT.md`
- `.planning/phases/01-contracts-config-and-auth-foundation/01-RESEARCH.md`
- `AGENTS.md`
- `packages/db/src/client.ts`
- Representative service env/auth modules in `services/*/src/`
</context>

<must_haves>
## Must Haves

### Truths
- Preserve HTTP-only cookie sessions; do not introduce JWT auth.
- Production must reject missing, dummy, or development-marked secrets.
- Session validation must fail closed when a user cannot be proven.
- `@pcp/shared` must remain source-only; do not add a build or `dist` contract.

### Phase Success Criteria Covered
- Services reject unsafe production secrets at startup.
- Session validation behavior is shared or contract-compatible across services.
</must_haves>

<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Service boots in production with dummy secrets | High | Zod env schemas reject unsafe values when `NODE_ENV=production`. |
| Auth behavior drifts across services | High | Shared session validation helper/contract and representative tests. |
| Auth helper returns identity but routes still miss tenant scope | Medium | Return typed user context now; leave full query audit to Phase 2. |
| Secrets leak in validation errors | High | Error messages identify variable names, not secret values. |
</threat_model>

<tasks>
## Tasks

<task id="1" type="analysis">
Inventory current env and auth patterns across all seven services. Identify
which services already have `env.ts`, `auth.ts`, route-local session reads, and
production secret guards. Do not modify files during this inventory.
</task>

<task id="2" type="implementation">
Create or adapt a shared session validation contract/helper in the narrowest
location that fits existing imports. The helper must validate the existing
cookie-backed session, return typed user context, and fail closed for missing,
expired, or invalid sessions.
</task>

<task id="3" type="implementation">
Replace duplicated session-validation code in representative services with the
shared contract. Prioritize services that already expose protected routes and
avoid broad unrelated route rewrites.
</task>

<task id="4" type="implementation">
Add Zod env validation to services that still rely on raw `process.env` fallback
values. Follow the `packages/db/src/client.ts` pattern. Production must reject
development markers for secrets.
</task>

<task id="5" type="test">
Add focused tests proving unsafe production env values fail startup validation
and invalid sessions are rejected. Keep tests per-package and respect the
existing Vitest version split.
</task>
</tasks>

<verification>

</verification>

# Plan 02-03: Add Tenant Isolation Regression Tests

## Objective

Convert the Phase 2 isolation contracts into durable tests that fail when tenant
predicates, tenant-prefixed storage contracts, or safe audit rules regress.

## Target Files

- Existing service tests under `services/workspace/src`
- Existing service tests under `services/runtime/src`
- Existing service tests under `services/publish/src`
- Existing service tests under `services/agent/src`
- Shared test helpers if a small helper reduces duplication without increasing
  coupling.

## Must Haves

- Tests include both owner-positive and cross-tenant-negative cases.
- Tests do not require Docker, Postgres, Redis, or MinIO.
- Tests respect the repo's split Vitest versions.
- Tests cover storage/path/label contracts without launching containers.
- No broad fixture framework unless existing local patterns already support it.

## Test Matrix

| Area                      | Contract                                                           |
| ------------------------- | ------------------------------------------------------------------ |
| Workspace datasets        | Cross-tenant dataset ids cannot be read, updated, or soft-deleted. |
| Workspace files/snapshots | Storage keys include tenant identity and reject traversal inputs.  |
| Runtime lifecycle         | Cross-tenant runtime ids cannot be observed or mutated.            |
| Runtime Docker config     | Tenant/resource labels are present in generated config.            |
| Publish lifecycle         | Cross-tenant hosted service ids cannot be observed or mutated.     |
| Publish Docker config     | Tenant/resource labels are present and env secrets are not logged. |
| Agent channels            | Channel polling cannot reveal another user's task state.           |
| Audit events              | Privileged event details are allow-listed and low-sensitivity.     |

## Tasks

1. Map critical methods touched in 02-01 and 02-02 to colocated tests.
2. Fill owner-positive and cross-tenant-negative tests for DB-backed flows.
3. Fill path/label/audit contract tests that run without external services.
4. Remove or simplify brittle mocks introduced during earlier plan work.
5. Run the full service test suite and typecheck.
6. Update Phase 2 summaries and requirement status after execution.

## Verification Commands

```powershell
pnpm test
pnpm typecheck
pnpm lint
```

## Done When

- Representative Phase 2 invariants are covered by automated tests.
- Removing a tenant predicate or tenant prefix causes at least one focused test
  to fail.
- GSD summaries and requirement statuses reflect completed Phase 2 coverage.

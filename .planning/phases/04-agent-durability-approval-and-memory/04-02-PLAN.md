# Plan 04-02: Add Agent Streaming And Token Telemetry

## Objective

Expose task progress through a realtime-compatible stream while persisting
provider/model/token/latency telemetry for agent runs.

## Target Files

- `services/agent/src/orchestrator.ts`
- `services/agent/src/routes.ts`
- `services/agent/src/llm/*`
- `packages/shared/src/agent.ts`
- frontend API/client surfaces if needed
- Phase 4 GSD summary/state artifacts

## Must Haves

- Existing polling APIs remain compatible.
- Task step creation can be streamed over SSE or WebSocket.
- LLM response usage is aggregated into `tasks.metadata`.
- Provider/model/latency metadata is persisted without secrets.
- Tests cover streaming route auth and telemetry persistence.

## Verification Commands

```powershell
pnpm --filter @pcp/agent-service test
pnpm test
pnpm typecheck
pnpm lint
```

## Done When

- AGT-03 and AGT-04 have service-level regression coverage.
- Frontend can keep polling or opt into stream events without API breakage.

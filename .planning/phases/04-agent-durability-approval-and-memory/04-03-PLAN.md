# Plan 04-03: Add Memory Indexing And Retrieval Improvements

## Objective

Make memory search production-ready with an intentional pgvector index,
documented retrieval strategy, and regression tests.

## Target Files

- `packages/db/src/schema/*`
- `packages/db/src/migrations/*`
- `services/memory/src/service.ts`
- `services/memory/src/service.test.ts`
- `services/memory/README.md`
- Phase 4 GSD summary/verification artifacts

## Must Haves

- pgvector index migration is intentionally owned and documented.
- Retrieval behavior is scoped by user/workspace metadata where applicable.
- Similarity threshold, limit, and ordering behavior are documented.
- Memory service tests cover retrieval behavior and tenant scoping.
- Existing dirty memory files are reviewed before staging to avoid accidental
  adoption of unrelated work.

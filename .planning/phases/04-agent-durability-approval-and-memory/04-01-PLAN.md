# Plan 04-01: Harden Tool Approval And Task Recovery Semantics

## Objective

Make approval-required tool calls durable, expiring, scoped, and auditable, and
make process restart recovery explicit so side-effectful work is not silently
lost or double-executed.

## Target Files

- `services/agent/src/orchestrator.ts`
- `services/agent/src/orchestrator.test.ts`
- Phase 4 GSD summary/state artifacts

## Must Haves

- Approval-required tools create linked `approval_requests` rows with
  `expiresAt`.
- Approval decisions are scoped by `taskId`, `userId`, and pending tool call.
- Expired approvals cannot execute side effects.
- Approval request, approval decision, expiry, and recovery transitions emit
  audit rows without plaintext secrets.
- Startup/recovery handling marks abandoned `executing` tasks and `running`
  tool calls failed instead of replaying them silently.

## Done When

- AGT-01 has durable approval request coverage.
- AGT-02 has explicit no-silent-replay recovery coverage.

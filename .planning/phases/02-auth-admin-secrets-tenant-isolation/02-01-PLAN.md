# Plan 02-01: Auth Admin And Provider Secret Hardening

<objective>
Make auth admin policy fail closed, remove PII from auth audit/log details, and ensure provider credential responses never expose encrypted secret material.
</objective>

<tasks>

<task type="auto">
<name>Add fail-closed admin policy</name>
<files>services/auth/src/env.ts, services/auth/src/routes/admin.ts</files>
<action>Add `ADMIN_EMAIL` to validated auth env. Update admin route helper so missing `ADMIN_EMAIL` returns 403 for authenticated users. Keep 401 for missing/invalid sessions. Use typed Fastify request/reply parameters and avoid raw `process.env` in the route.</action>
<verify>pnpm --filter @pcp/auth-service typecheck</verify>

<task type="auto">
<name>Map provider credential DTOs explicitly</name>
<files>services/auth/src/routes/profile.ts</files>

<verify>pnpm --filter @pcp/auth-service typecheck</verify>
</task>

</task>
